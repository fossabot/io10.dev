Privacy Policy for Access to Google APIs

io10.dev is using the Google Drive API to make it easier to store files on Google Drive. io10.dev uses the io10.dev Application on Google Cloud Platform. (That is the name you will see in a consent screen.) This integration is governed by common policies, recorded here. Your use of Google APIs with io10.dev is subject to each API's respective terms of service. See Google APIs Terms of Service.

Accessing user data

io10.dev accesses Google resources from the machine where you use io10.dev. Your machine communicates directly with the Google APIs. The io10.dev Application never receives your data or the permission to access your data. The owners of the project can only see anonymous, aggregated information about usage of tokens obtained through its OAuth client, such as which APIs and endpoints are being used. io10.dev sends users to https://accounts.google.com/o/oauth2/auth for authorization and does not directly access or collect user data used by Google Auth.

Using user data

io10.dev includes functions that you can execute in order to read or modify your own data (or data shared with you). This can only happen after you provide a token, which requires that you authenticate yourself as a specific Google user and authorize these actions. io10.dev can help you get a token by guiding you through the OAuth flow in the browser. There you must consent to allow the io10.dev Application to operate on your behalf. The OAuth consent screen will describe the scope of what is being authorized e.g. it will name the target API(s) and whether you are authorizing “read only” or “read and write” access. There are two ways to use io10.dev with Google Drive without authorizing the io10.dev Application: bring your own service account token or configure the package to use an OAuth client of your choice. See Setup a Google Drive io10.dev Remote for more information.

Scopes

userinfo.email scope (view your email address)' userinfo.profile scope (see your personal info. including any personal info you've made publicly available), and openid scope (authenticate using OpenID Connect) are required to be used by Google Auth. io10.dev allows you to manage your Google Drive files and therefore the default scopes also include drive (read/write access to your Google Drive) and drive.appdata (manage configuration folder in your Google Drive).

Sharing user data

io10.dev only communicates with Google APIs. No user data is shared with the owners of the io10.dev or any other party.

Storing user data

io10.dev stores your credentials on your machine, for later reuse by you. Use caution when using Google Drive io10.dev remotes on shared machines.

Usage in other packages or applications

Do not use an API key or client ID from the io10.dev Application in an external package or tool. Per the Google User Data Policy, your application must accurately represent itself when authenticating to Google API services.
